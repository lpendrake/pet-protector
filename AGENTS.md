# Pet Protector Context

## Description

The general concept is a simple game that can run in a terminal where the player is looking after a pet. 
We'll start off simple and it just needs things like feeding, watering, petting, playing with, etc.

Later we'll add in more complex logic like events that happen to shake things up.

One key feature is that the timeline. This is a passive game. You take a few actions a day and simply need to remember to come back and check on your pet. 

This means that if the player opens the game after 3 days (or 3 months) we need to simulate the passage of time since they last checked on it.
That simulation of the passage of time should have the same end result as if the player had sat there with the terminal open the whole time.

## Initial Visuals

No graphic of the pet. Just display the stats of the pet and the actions the player can take.
Later we will likely move this to a web interface that opens in a browser but the game will never be a 3d game.
Cute pixel graphics is the likely end result.
